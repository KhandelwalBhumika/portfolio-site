What is a URL?

A URL, or Uniform Resource Locator, is effectively a unique web address. It represents the ‚Äúlocation‚Äù of a specific resource on the internet. The term URL refers to a subset of Uniform Resource Indicators (URIs) that, in addition to identifying the web resource‚Äîalso provides the means of locating it. Every resource on the internet has a URL including: webpages, files, images, media, web applications, services, etc.

Path:

The path refers to the exact location of a page, post, file, or other asset. It is often analogous to the underlying file structure of the website. The path resides after the hostname and is separated by ‚Äú/‚Äù (forward slash). The path/file also consists of any asset file extension, such as images (.jpg or .png, etc.), documents (.pdf or .docx), and more. But not all URLs will display a path.For example:

Query String / Parameters:

Parameters are found at the very end of the URL or within the path, depending on the implementation. URL Parameters are represented in key/value pairs, beginning with a '?' and separated by an ampersand '&'. They can also be dynamically set in the path as values separated by slashes and other characters (depending on the system being used and how it's implemented). Parameters are commonly used for tracking and analytics‚Äîas well as encoding specific information for use within websites and applications.For example, a URL with parameters:

Fragment:

Any URL that contains a # character is a fragment URL. The portion of the URL to the left of the # identifies a resource that can be downloaded by a browser and the portion on the right, known as the fragment identifier, specifies a location within the resource:

  • http://example.com/?color=#ffff&shape=circle
  • http://www.httpwatch.com/features.htm#print
